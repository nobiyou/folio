# Folio 内容模型指南

Folio 现已完全依赖 WordPress 默认文章（`post`）来承载作品与博客内容。本指南帮助你理解新的内容结构、迁移旧数据，并为团队协作提供可复用的操作步骤。

## 1. 核心原则

- **单一内容类型**：所有「作品」「项目」「博客」都使用 `post`，避免额外的自定义文章类型（CPT）带来的兼容性与迁移负担。
- **分类映射作品类型**：通过文章分类（`category`）与标签（`post_tag`）区分不同作品系列或题材，例如 `作品类型 / Branding`、`行业 / 游戏`。
- **自定义字段记录细节**：若需要客户名称、完成日期、项目链接，可使用自定义字段（ACF 或主题内置 Meta Box）而非新建 CPT。
- **统一展示层**：`front-page.php`、`archive.php`、`template-parts/content-post.php` 均假设来自 `post`，并通过缓存/权限系统加速。

## 2. 推荐分类结构

| 目的            | 建议做法                                               |
|-----------------|--------------------------------------------------------|
| 作品类型        | 在主分类中创建「案例」「插画」「UI/UX」等一级分类。     |
| 行业/主题       | 使用标签快速标记行业或关键词，方便多重筛选。           |
| 作品状态/特色   | 可利用自定义字段（如 `folio_featured`）或标签（`精选`）。|

> **提示**：保持分类深度 <= 2 层，方便导航与面包屑渲染；标签数量控制在 10 以内，便于搜索。

## 3. 发布流程

1. **撰写文章**：后台进入「文章 > 新建文章」，填写标题、内容、特色图片。
2. **选择分类**：为文章勾选对应作品类别与行业标签。
3. **额外元数据**（可选）：通过主题设置或自定义字段录入客户/链接等信息。
4. **会员权限**：如需限制 VIP/SVIP，使用 `inc/class-premium-content-enhanced.php` 的短代码或文章元框。
5. **缓存与统计**：发布或更新文章时首页缓存自动失效；阅读数/点赞/收藏由 `folio_Post_Stats` 负责，无需手动处理。

## 4. 迁移旧 Portfolio 数据

若早期版本曾使用 `portfolio` CPT，可按以下步骤迁移：

1. **导出数据**：进入「工具 > 导出」选择 `portfolio`，保存 XML。
2. **导入到文章**：在导出的 XML 中将 `<wp:post_type>portfolio</wp:post_type>` 替换为 `<wp:post_type>post</wp:post_type>`（或使用脚本转换），再通过「工具 > 导入」执行。
3. **映射分类**：将原 `portfolio_category` 名称映射到现有文章分类，可使用分类迁移插件或 WP-CLI。
4. **修正模板**：确认 `template-parts/content-post.php` 已能展示所需字段；如有特殊内容，使用自定义字段存储。
5. **清理遗留**：删除未使用的小工具/菜单项，并确认主题或插件中不再引用 `portfolio` CPT。

## 5. 搜索引擎与访问控制

- 分页已向访客开放，主题会对可疑爬虫进行 429 节流；无需再通过登录墙阻断。
- 内容访问日志（`folio_Security_Protection_Manager`）对普通浏览进行了采样，减少数据库压力。
- 阅读数统计通过延迟的 AJAX + Cookie 机制记录，兼容前端缓存。

## 6. FAQ

- **Q: 如何区分「作品」与「博客」？**  
  A: 建议建立 `作品`、`博客` 两个顶级分类，在菜单或查询中使用 `category_name` 参数筛选。

- **Q: 需要单独的作品 URL 结构吗？**  
  A: 可通过 `category` 前缀实现，例如 `/portfolio/%postname%/`；无需新增 CPT。

- **Q: 还能将作品导入第三方展示站吗？**  
  A: 可以——所有内容均为 `post`，可使用 WordPress API/Feed 提取，或在 `functions.php` 中增加自定义 REST 端点。
